Naming Convention: Three rules to follow when naming files for the web

All lowercase: Because different operating systems treat capitalization differently. No spaces! Because URLs cannot have spaces in them. Only letters, numbers, and dashes (-) Because URLs use a limited set of characters & dashes are word separators while underscores are word connectors.

Best Practices for Commit Messages

  • Minimum 3 words & 10 characters
  • Proper spelling & grammar
  • Capital starting letter, no period
  • Must start with an imperative verb: Create, Fix, Add, Solve, etc.
  • Pretend every commit starts with the phrase: “This commit will…”

Header

The header element is used to define a header section for a document or section. It often contains a logo, navigation links, and other important information.

Nav

The nav element is used to define a section of a document that contains navigation links. It is typically used for site navigation menus, table of contents, and other navigational blocks.

Main

The main element is used to define the main content of a document or section. It should not include any content that is repeated across multiple pages, such as navigation links or headers.

Section

The section element is used to define a section of a document or page. It is typically used to group related content together, such as articles or blog posts.

Article

The article element is used to define an independent, self-contained block of content, such as a blog post, news article, or product review.

Div

The div element is a generic container used to group content together, typically for styling or scripting purposes. It should be used sparingly and only when there is no other appropriate semantic element.

Aside

The aside element is used to define a section of a document that is related to the main content, but not essential to the overall meaning of the document. It is often used for sidebars, callouts, or related links.

Footer

The footer element is used to define a footer section for a document or section. It often contains copyright information, contact details, and other related information.

Span

The span element is a generic inline container used to group content together, typically for styling or scripting purposes. It should be used sparingly and only when there is no other appropriate semantic element.

Small

The small element is used to indicate that the text inside it is of lesser importance or smaller font size than the surrounding text. It is often used for disclaimers, copyright notices, or other minor details.
